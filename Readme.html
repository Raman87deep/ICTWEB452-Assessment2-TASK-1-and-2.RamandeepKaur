How to ensure CSS works on Google (GitHub Pages or any static host)

  1. Upload all files to the same folder (root). The HTML pages already reference the CSS via <link rel="stylesheet" href="styles.css">.
  2. Make sure filenames are exactly these (case‑sensitive on most hosts): styles.css, script.js, and the four .html pages.
  3. Do not place files inside subfolders unless you also update the paths in the HTML.

Quick local test

Why CSS may not show when opened via Google Drive/Docs preview

Opening .html files directly in Google Drive/Docs uses a preview sandbox that often blocks loading sibling files like styles.css. This is a platform limitation, not an issue with your code. To see your CSS, use one of these options:

  • Host the folder on a static host (e.g., GitHub Pages, Netlify, Vercel).
  • Use Google Sites (rebuild pages in Sites) rather than Drive preview.
  • Run a local server: python -m http.server 5500 and open http://127.0.0.1:5500/index.html.
